# Author Voice Template

只在需要把内容写得更像作者本人时读取本文件。

## 目标

从用户提供的真实样本、口头要求或当前对话里，提炼出一个可执行的作者口吻模板，然后在回写阶段使用它。

## 默认做法

如果用户没有提供明确样本：
- 保持第一人称视角
- 优先真实、克制、有思考感的表达
- 不装权威
- 不要写成营销文或自媒体套路文
- 允许保留“我还没完全想清楚，但我现在越来越觉得……”这种探索感

如果用户提供了样本：
- 先观察句子长短、段落密度、开头方式、收尾方式
- 观察用户是先讲背景，还是先抛判断
- 观察用户是否喜欢把问题拆层
- 观察用户是否接受较长展开，还是偏爱极短表达
- 再生成一个临时口吻模板，用于本轮和后续回写

## 推荐提炼维度

优先总结这些维度：
- 语气：克制 / 直接 / 温和 / 强结论
- 视角：第一人称 / 观察者 / 解释者
- 结构：先背景后判断 / 先判断后展开 / 拆层推进
- 节奏：短句密集 / 长短混合 / 偏完整段落
- 禁忌：太像文章 / 太像 AI / 太像营销 / 太像行业评论

## 改写原则

- 保留作者原本的判断和立场，不替作者换观点。
- 口吻回写不是简单“口语化”，而是更像作者本人会发出来。
- 用户内容已经清楚时，再做口吻回写；不要在概念没清楚时过早修辞。
- 如果用户的样本明显更适合长一点的思考型表达，不要硬压成极短爆款风格。

## 默认输出

当用户要求“像我一点”时，优先给：
- 一版更像作者本人会说的话的版本
- 需要时再补一版更短的动态版
