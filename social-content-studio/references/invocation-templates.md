# Invocation Templates

只在需要给用户一组可以直接复制使用的标准调用方式时读取本文件。

## 1. 深聊模板

适合：
- 我觉得这条笔记里有几个关键点，但我没想透
- 先不要写文章，先把概念拆清楚

模板：

```text
Use $social-content-studio 先不要帮我写文章，先把这段笔记里的核心问题拆出来，指出我已经想清楚和没想清楚的部分，再问我一个最关键的问题。
```

加强版：

```text
Use $social-content-studio 我觉得这段内容里有几个重要判断，但我还没想透。先帮我拆概念、找模糊点、给我一个更深一层的视角，然后再决定要不要整理成内容。
```

## 2. 结构化模板

适合：
- 我有一些零碎想法
- 我不想现在写成文，只想先整理成清楚结构

模板：

```text
Use $social-content-studio 把这堆碎片想法整理成一份观点骨架，不要扩写成长文。请区分我真正想说的、已经明确的判断、还没想清楚的部分，以及可以继续展开的方向。
```

针对单条笔记：

```text
Use $social-content-studio 把这条笔记整理成一份观点骨架，先不要写成文。我要看到核心问题、已明确判断、仍模糊的部分和下一步最值得想的问题。
```

## 3. 草稿装配模板

适合：
- 我已经有观点骨架或半成品草稿
- 我想把它变成可发布草稿

模板：

```text
Use $social-content-studio 基于这份观点骨架，帮我装配成一版可发布草稿，保留我的语气，不要过度扩写。
```

多平台版：

```text
Use $social-content-studio 基于这份观点骨架，帮我整理成公众号版和小红书版草稿，并补上标题、摘要、标签和审核备注。
```

## 4. 一条龙模板

适合：
- 从碎片笔记开始
- 先深聊，再结构化，最后出草稿

模板：

```text
Use $social-content-studio 先跟我深聊，把这段笔记真正想说的问题拆出来；等我想清楚后，再整理成观点骨架，最后帮我装配成一版可审核的社媒草稿。
```

## 使用提醒

- 如果你还没想透，就优先用“深聊模板”。
- 如果你已经想得差不多了，但内容很散，就用“结构化模板”。
- 如果你已经有骨架或初稿，就用“草稿装配模板”。
- 不要一上来默认要求它写完整文章，除非你真的已经想清楚。
